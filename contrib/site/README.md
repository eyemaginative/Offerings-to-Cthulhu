# 23skidoo.info — static site snapshot

Hand-authored static content for `https://23skidoo.info/`, the public landing
page + lore page for Offerings to Cthulhu (OFF). Versioned in this repo so the
copy survives a the production host disk loss; the live site is served from
`/var/www/23skidoo.info/` on the production host.

## What's here

| Path | What it is |
|---|---|
| `index.html` | Homepage. SubGenius/Cthulhu parody-religion lore + chain spec + countdown link + Codex link. Hand-edited; iterate freely. |
| `ritual.html` | The Ritual lore page, rebuilt from a 2015 Wayback mirror into the current dark/orange theme. |
| `static/css/` | Stylesheets shared across static pages. |
| `static/font/` | Embedded web fonts. |
| `static/img/` | Logos, mascot art (`off_240x240.png`, `cthulhu-bg.jpg`, etc.). The squid logo appears in headers/footers across all pages including the codex e-reader and awakening countdown. |
| `static/js/` | Frontend JS (if any — most pages are server-rendered). |
| `static/corpus_v1.bin` | Frozen Lovecraft corpus (2.2 MB, SHA-256 `1c808560…`). The OFF binary references this hash in `chainparams.cpp::kCorpusSha256`. Also lives at `contrib/codex/codex_corpus.txt` (same bytes, different filename — the website serves this one for download). |
| `static/corpus_v1.bin.sha256` | Hash file matching the canonical kCorpusSha256. |
| `static/corpus_v1.mirrors.json` | Public mirror list for fetching the corpus. |

## What's NOT here

| Path | Why |
|---|---|
| `static/tts/` (121 MB) | Generated MP3s, content-addressed cache rendered by `contrib/codex/build_tts.py`. Gitignored — regenerable from the canon. |
| `codex/` | Generated by `contrib/codex/build_ereader.py` cron (every 2m). Output dir; not versioned. |
| `awakening/` | Generated by `contrib/codex/build_countdown.py` cron (every 1m). Output dir; not versioned. |
| `bridge/` | Reclamation portal — separate FastAPI app at `/opt/bridge-portal/` with its own deploy lifecycle. Includes an indexer + creds that should NOT be in this repo. |
| `index.html.bak-*`, `index.html.old-*` | Historical local backups; not relevant to canonical history. |

## Workflow

**Current state (as of 2026-05-25):** this is a **snapshot for recovery**. The live files at `/var/www/23skidoo.info/` are still the editable source on the production host; edits there are NOT automatically synced back to this directory.

To re-sync after a manual edit to the live site:
```bash
cd /home/btcbob/claude/offerings-master
cp /var/www/23skidoo.info/index.html /var/www/23skidoo.info/ritual.html contrib/site/
git add contrib/site && git commit -m "site: sync hand-edits from live" && git push
```

**Alternative workflow (not yet enabled):** symlink the live paths to this directory so all live edits go through git automatically:
```bash
# Atomic swap with backup
sudo mv /var/www/23skidoo.info/index.html /var/www/23skidoo.info/index.html.pre-symlink
sudo ln -s /home/btcbob/claude/offerings-master/contrib/site/index.html /var/www/23skidoo.info/index.html
# Repeat for ritual.html and the static/ subdirs (excluding tts/)
```
Nginx follows symlinks by default; no config change needed. Trade-off: every edit becomes a commit. Discuss before flipping.
